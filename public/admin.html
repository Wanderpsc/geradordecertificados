🔐 Autenticação (JWT)

Fluxo de Login

POST /api/auth/login
Body: { email, senha }

→ Resposta:
{
  token: "eyJhbGc...",   // JWT 30d
  usuario: {
    id, nome, email,
    role: "admin"|"user",
    licenca: { tipo, status, dataExpiracao }
  }
}

Uso do Token

// Header em todas as rotas protegidas:
Authorization: Bearer <token>

// Middleware proteger() verifica:
1. Presença do header Authorization
2. Validade do JWT (JWT_SECRET)
3. Existência do usuário no banco
4. Se ativo: true

// Middleware apenasAdmin():
→ Verifica role === "admin"

// SubUsuário:
→ JWT com tipo: "subUsuario"
→ Herda licença e escola do dono

💳 Fluxo de Pagamentos (Mercado Pago)

PIX

1. Cliente escolhe plano
2. POST /api/pagamentos/pix → cria Pagamento(pendente) + chama MP SDK
3. Frontend exibe QR Code (base64)
4. MP chama POST /api/pagamentos/webhook
5. Webhook verifica status === "approved"
6. Pagamento → aprovado, chama provisionarLicenca()
7. Licença criada/renovada no banco

Cartão (Checkout Pro)

1. POST /api/pagamentos/cartao → cria Preference no MP
2. Frontend redireciona para MP Checkout
3. MP processa pagamento
4. Redirect para back_url success/failure/pending
5. Frontend chama POST /api/pagamentos/verificar-aprovacao
6. Busca MP Payment mais recente do usuário
7. Se aprovado → provisiona licença
8. Webhook também pode confirmar o pagamento
ISS: Em modo produção, aplica-se 5% de ISS (Simples Nacional, LC 116/2003, código 1.07 — software). Alíquota configurada em DADOS_PRESTADOR.aliquotaISS (admin.js) e exibida na seção Financeiro.
